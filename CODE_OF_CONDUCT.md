# Code of Conduct

This project is intended to support careful, evidence-oriented technical discussion.

## Expected behavior

- Be respectful and professional.
- Critique specifications, examples, fixtures, and implementations rather than people.
- Distinguish normative specification text from non-normative notes.
- Keep disagreements grounded in concrete examples, compatibility concerns, safety concerns, or implementation evidence.
- Help maintain the CAP-Digest / CAP-Core boundary.

## Unacceptable behavior

- Personal attacks or sustained disruptive behavior.
- Publishing private information without permission.
- Repeatedly moving issues or pull requests away from their stated scope.
- Presenting non-normative notes as accepted specification.
- Pressuring maintainers to merge security-sensitive changes without review.

## Enforcement

Maintainers may edit, hide, lock, or close discussions that violate this code. Maintainers may also ask contributors to move broad design discussion into a CAPP or a non-normative note.

## Scope

This code applies to project spaces including issues, pull requests, discussions, commits, and documentation contributions.
